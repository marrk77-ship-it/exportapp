## 管理者の役割

管理者は以下の業務を担当します：

- ✅ **ユーザー管理**: 新規ユーザーの追加、削除、パスワードリセット
- ✅ **データ管理**: ユーザーのCSVデータの確認、削除
- ✅ **監視**: システム利用状況の確認、操作ログの監視
- ✅ **サポート**: ユーザーからの問い合わせ対応

**URL**: https://exportapp-tw.tech/admin

### ログイン情報

- **パスワード**: `admin2024`

## ユーザー管理

### ユーザー一覧の表示

**「👥 ユーザー管理」ボタンをクリック**すると、登録されている全ユーザーが表示されます。

#### 表示項目

| 項目 | 説明 |
|------|------|
| **ログインID** | ユーザーのログインID |
| **名前** | ユーザーの表示名 |
| **権限** | admin（管理者）または user（一般ユーザー） |
| **CSV件数** | 保存されているCSVデータの行数 |
| **登録日** | アカウント作成日時（日本時間） |
| **操作** | 各種操作ボタン |

#### 操作ボタン

| ボタン | 機能 | 説明 |
|--------|------|------|
| 📋 **CSV** | CSVプレビュー | ユーザーのCSVデータを確認 |
| 📥 **履歴** | アップロード履歴 | ユーザーのアップロード履歴を確認 |
| 🗑️ **削除** | CSV削除 | ユーザーのCSVデータを完全削除 |
| 🔑 **リセット** | パスワードリセット | パスワードを初期化 |
| ❌ **削除** | ユーザー削除 | ユーザーアカウントを削除 |

---

### 新規ユーザーの追加

#### 手順

1. **「➕ 新規ユーザー追加」ボタンをクリック**

2. **以下の情報を入力**
   - **ログインID**: 半角英数字（例: `client3`）
   - **パスワード**: 8文字以上推奨（例: `password789`）
   - **名前**: ユーザーの表示名（例: `株式会社サンプル`）
   - **権限**: `user`（一般ユーザー）または `admin`（管理者）

3. **「追加」ボタンをクリック**

4. **確認メッセージが表示される**
   - 成功: 「ユーザーを追加しました」
   - 失敗: エラーメッセージが表示される

5. **ユーザーに情報を伝える**
   - ログインID
   - 初期パスワード
   - システムURL: https://exportapp-tw.tech

#### ⚠️ 注意事項

- ログインIDは既存のIDと重複できません
- パスワードは必ず安全な方法でユーザーに伝えてください（メール、電話など）
- 初回ログイン後、ユーザーにパスワード変更を推奨してください

---

### パスワードリセット

ユーザーがパスワードを忘れた場合に使用します。

#### 手順

1. **対象ユーザーの「🔑 リセット」ボタンをクリック**

2. **確認ダイアログが表示される**
   ```
   [ユーザー名] のパスワードを「password123」にリセットしますか？
   ```

3. **「OK」をクリック**

4. **パスワードがリセットされる**
   - 新しいパスワード: `password123`

5. **ユーザーに新しいパスワードを伝える**

#### ⚠️ セキュリティ注意

- リセット後のパスワードは全ユーザー共通の `password123` です
- ユーザーには早急にパスワード変更を指示してください
- パスワードは安全な方法で伝達してください

---

### ユーザーの削除

#### 手順

1. **対象ユーザーの「❌ 削除」ボタンをクリック**

2. **確認ダイアログが表示される**
   ```
   本当に [ユーザー名] を削除しますか？
   この操作は取り消せません。
   ```

3. **「OK」をクリック**

4. **ユーザーとそのデータが完全に削除される**
   - ユーザーアカウント
   - CSVデータ
   - アップロード履歴

#### ⚠️ 重要な注意

- **削除は取り消せません**
- **全てのデータが永久に削除されます**
- 削除前に必要なデータはバックアップしてください
- 自分自身（現在ログイン中の管理者）は削除できません

## CSVデータ管理

### CSVデータのプレビュー

各ユーザーがアップロードしたCSVデータを確認できます。

#### 手順

1. **対象ユーザーの「📋 CSV」ボタンをクリック**

2. **CSVプレビューモーダルが表示される**
   - デフォルトで最初の10件が表示されます
   - 最大1000件まで表示可能

3. **表示件数を変更する場合**
   - 「表示件数」のドロップダウンで選択
     - 10件
     - 50件
     - 100件
     - 500件
     - 1000件

4. **データを確認**
   - 列名、データ内容、行数を確認できます

5. **「閉じる」ボタンでモーダルを閉じる**

#### 表示される情報

- **ファイル名**: アップロードされたCSVファイル名
- **行数**: データの総行数
- **列名**: CSVの各列の項目名
- **データ**: 実際のデータ内容

---

### アップロード履歴の確認

各ユーザーの過去のアップロード履歴を確認できます。

#### 手順

1. **対象ユーザーの「📥 履歴」ボタンをクリック**

2. **アップロード履歴モーダルが表示される**

3. **履歴情報が時系列で表示される**
   - ファイル名
   - 行数
   - アップロード日時（日本時間）

4. **履歴からダウンロード可能**
   - 各履歴の「⬇️」ボタンをクリック
   - 過去にアップロードされたCSVファイルがダウンロードされます

#### 表示例

```
アップロード履歴 - client1

ファイル名                              | 件数  | アップロード日時        | 操作
電子マニフェスト_2026-03-25.csv         | 476行 | 2026/03/25 15:30:45    | ⬇️
電子マニフェスト_2026-03-20.csv         | 450行 | 2026/03/20 10:15:30    | ⬇️
```

---

### CSVデータの削除

ユーザーのCSVデータとアップロード履歴を完全に削除します。

#### 手順

1. **対象ユーザーの「🗑️ 削除」ボタンをクリック**

2. **確認ダイアログが表示される**
   ```
   [ユーザー名] のCSVデータを全て削除しますか？
   ```

3. **「OK」をクリック**

4. **以下が削除される**
   - ユーザーのCSVデータ（全行）
   - アップロード履歴（全件）

5. **削除後、データ件数が「0行」になる**

#### ⚠️ 注意事項

- 削除は取り消せません
- ユーザーアカウント自体は削除されません（データのみ削除）
- ユーザーは再度CSVをアップロード可能です

## 操作ログの確認

全ての管理操作は自動的にログに記録されます。

### ログ画面の表示

**「📋 操作ログ」ボタンをクリック**すると、操作ログ画面が表示されます。

### 表示項目

| 項目 | 説明 |
|------|------|
| **日時** | 操作が実行された日時（日本時間） |
| **管理者** | 操作を実行した管理者のログインID |
| **操作** | 実行された操作の種類 |
| **詳細** | 操作の詳細情報 |
| **IPアドレス** | 操作元のIPアドレス |

### 記録される操作

| 操作タイプ | 説明 | 詳細例 |
|-----------|------|--------|
| `create_user` | ユーザー作成 | `ユーザー「client3」を作成` |
| `delete_user` | ユーザー削除 | `ユーザー「client2」を削除` |
| `reset_password` | パスワードリセット | `ユーザー「client1」のパスワードをリセット` |
| `delete_csv` | CSV削除 | `ユーザー「client1」のCSVデータ（476行）を削除` |

### ログの活用方法

1. **セキュリティ監査**
   - 不正なアクセスや操作がないか確認
   - IPアドレスで操作元を特定

2. **トラブル調査**
   - 問題発生時の操作履歴を確認
   - いつ、誰が、何をしたか追跡

3. **業務記録**
   - 管理業務の記録として保管
   - 定期的な確認と報告

### ログの保持期間

- ログは永続的に保存されます
- データベースの容量に応じて定期的なバックアップを推奨します

## セキュリティ管理

### 🔒 セキュリティベストプラクティス

#### 1. パスワード管理

- ✅ デフォルトのadminパスワード（`admin2024`）を必ず変更
- ✅ 強力なパスワードを使用（大文字、小文字、数字、記号の組み合わせ）
- ✅ 定期的なパスワード変更（3ヶ月〜6ヶ月ごと）
- ✅ パスワードを他人と共有しない

#### 2. アクセス管理

- ✅ 管理者権限は最小限の人数に付与
- ✅ 退職者のアカウントは即座に削除
- ✅ 長期間使用されていないアカウントは定期的に確認

#### 3. ログ監視

- ✅ 操作ログを定期的に確認（週1回推奨）
- ✅ 不審なIPアドレスからのアクセスがないか確認
- ✅ 異常な操作パターンがないか監視

#### 4. データ保護

- ✅ 重要なデータは削除前にバックアップ
- ✅ ユーザー情報の取り扱いに注意
- ✅ データの不正持ち出しがないか監視

---

### 🚨 セキュリティインシデント対応

#### 不正アクセスの疑いがある場合

1. **該当ユーザーのアカウントを即座に停止**
   - パスワードをリセット
   - または一時的にアカウント削除

2. **操作ログを確認**
   - 不正な操作がないか調査
   - IPアドレスを記録

3. **影響範囲を調査**
   - 削除されたデータがないか確認
   - 不正にダウンロードされたデータがないか確認

4. **パスワードを変更**
   - 影響を受けた可能性のある全アカウントのパスワードを変更

5. **報告と記録**
   - インシデントの内容を記録
   - 必要に応じて上位者に報告

## トラブルシューティング

### ユーザーがログインできない

#### 原因と対処法

| 原因 | 対処法 |
|------|--------|
| パスワード忘れ | パスワードリセットを実行 |
| アカウントが存在しない | ユーザー一覧で確認、必要なら新規作成 |
| ログインIDの入力ミス | 正しいログインIDをユーザーに伝える |
| システム障害 | システムの稼働状況を確認 |

---

### CSVデータが正しく表示されない

#### 原因と対処法

| 原因 | 対処法 |
|------|--------|
| ファイル形式が不正 | ユーザーにCSV形式のファイルを再送信してもらう |
| 文字コードの問題 | UTF-8形式のCSVファイルを使用 |
| データが大きすぎる | ファイルサイズを確認、分割を検討 |

---

### スプレッドシート出力ができない

#### 原因と対処法

| 原因 | 対処法 |
|------|--------|
| CSVデータがアップロードされていない | ユーザーにCSVを再アップロードしてもらう |
| ネットワークエラー | ネットワーク接続を確認 |
| ブラウザの問題 | 別のブラウザで試す、キャッシュをクリア |

---

### 操作ログが表示されない

#### 原因と対処法

| 原因 | 対処法 |
|------|--------|
| ログが記録されていない | データベースの状態を確認 |
| ブラウザのキャッシュ | ブラウザをリフレッシュ（Ctrl+Shift+R） |
| 管理者権限がない | ログイン中のアカウントの権限を確認 |

---

### ユーザーから「詳細設定が上手くいかない」と連絡があった

#### よくある問題と対処法

**問題1: 列名を指定しても出力されない**

| 原因 | 対処法 |
|------|--------|
| 列名のスペルミス | CSVプレビューで正確な列名を確認してもらう |
| 全角・半角の違い | 完全一致が必要であることを説明 |
| カンマ区切りのスペース | `列1,列2,列3`（スペースなし）と指定する |

**問題2: フィルターが効かない**

| 原因 | 対処法 |
|------|--------|
| フィルター値が未入力 | フィルター対象列とフィルター値の両方が必要 |
| 列名が間違っている | CSVに実際に存在する列名か確認 |
| 該当データが存在しない | フィルター値を変更して再試行 |

**問題3: 設定が保存されない**

| 原因 | 対処法 |
|------|--------|
| 保存ボタンを押していない | 「💾 設定を保存」ボタンのクリックを確認 |
| ブラウザのキャッシュ | ページをリフレッシュして再度確認 |
| システムエラー | ブラウザコンソールでエラーを確認 |

#### 対応手順

1. **状況を詳しく聞く**
   - どの設定項目を変更したか
   - どのような結果になったか
   - エラーメッセージが表示されたか

2. **CSVプレビューで列名を確認**
   - 管理画面で該当ユーザーのCSVをプレビュー
   - 正確な列名をユーザーに伝える

3. **設定例を提示**
   - ユーザーマニュアルの「設定例」を参照してもらう
   - 具体的な設定値を提示

4. **テスト実行**
   - 一緒に設定を入力して動作確認
   - 結果を確認しながら調整
